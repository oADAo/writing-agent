# 每月遊戲推薦：使用者改稿邏輯

更新日期：2026-05-14

## 2026-05-14 每月遊戲完整交付流程

使用者明確指定：之後每月遊戲推薦要固定包含 `研究 / 撰寫 / 提交` 三段，不要只記其中一段。

### 研究

- 先做候選遊戲池，確認發售日期、平台、買斷制邊界與是否符合玫玫物語頻道。
- 每款都是獨立研究單位，不能只靠官方商店頁或新聞摘要。
- 每款研究卡至少要有：`已確認事實 / 核心玩法 / 最大賣點 / 社群期待與疑慮 / 適合誰 / 風險或未定點 / 可直接改寫口播句`。
- 使用者還沒確認名單前，預設交 `月份推薦研究素材包`，不要直接收斂正式稿。

### 撰寫

- 使用者確認名單後，依發售日期由早到晚排序。
- 正式稿套用全域 Maymei 中文撰寫系統與使用者雲端親改稿語氣。
- 每款先講「這款是什麼」，再講玩法上我們會做什麼，最後收在適合誰或一句自然期待。
- 正文不寫研究過程、選題理由、表格填寫邏輯或後台判斷。

### 提交

- 正式稿完成後，預設提交到每月固定 Drive 資料夾。
- 同步更新兩份雲端檔：
  - Google Docs：月份推薦文案正式稿。
  - Google Sheets：`Longform Episode Workbook Template` 的 `遊戲清單` 工作表。
- Sheets 必填欄位是：`遊戲名稱 / 遊戲宣傳片 / 遊戲實機影片 / 遊戲音樂 / 發售日期 / 平台`。
- 寫入後必須重新讀取 Docs / Sheets 目標內容確認成功，再回報完成。

## 2026-05-14 每月表格音樂欄修正

使用者明確提醒：每月遊戲推薦的 `Longform Episode Workbook Template` 音樂欄，不能因為本作音樂不足就直接空掉。要照固定規則往下找：

1. 先找本作音樂。
2. 本作找不到可用乾淨音樂時，找同系列前作。
3. 同系列也不行，再找同開發商或發行商之前作品。
4. 最後才找同類型替代。

寫入表格時要在 note 標明來源層級，例如 `本作音樂 / 系列前作 / 同公司舊作 / 同類型替代`，不要把替代曲誤寫成該款遊戲官方音樂。熱血西遊記這類明顯有系列脈絡的作品，應優先找 `熱血系列 / River City / Kunio-kun` 系列曲，不要直接留空。

寫入前仍要做音訊檢查：抓 00:20-01:00 或合適片段，確認沒有明顯人聲、歌詞、旁白、尖銳音效或過吵鼓點。若只能用 Whisper 或本地音訊檢查，工作記錄要寫清楚檢查方式。

## 2026-05-13 雲端最終稿學習

使用者明確指定：之後撰寫都要用使用者自己的中文撰寫。這版 `六月遊戲推薦` 雲端正式稿，是之後 `月份推薦 / 新作推薦 / 推薦清單` 的最高優先中文樣本。

### 這版最像使用者中文的地方

- 單款遊戲不用每段都塞滿研究資料，重點是先讓觀眾馬上知道「這款是什麼」。
  - 例：`這款是《熱血》系列的新作，這次直接跑去改編《西遊記》。`
  - 例：`這是一款舒服的生存建造遊戲。`
- 句子可以有資訊，但要一路往前推，不要重複換句話講同一件事。
- 一款遊戲通常 2 到 3 段就夠：
  1. 先講遊戲定位或最醒目的賣點。
  2. 再講玩法上我們會做什麼。
  3. 最後收在適合誰、為什麼可以留意。
- 遊戲介紹要像在幫觀眾挑片，不像在交研究報告。
- 可以自然加入個人感：
  - `這款遊戲我超久以前就放進了願望清單`
  - `算是我自己蠻期待的一款遊戲`
  但這種句子要少量使用，只放在真的有私心期待的作品。
- 收尾句要簡單自然：
  - `可以玩一下哦`
  - `可以留意一下這款哦`
  - `很值得注意`
  - `可以先記起來`
  - `算是我自己蠻期待的一款遊戲`

### 之後月推稿要照這樣寫

- 先寫「這款是什麼」，不要先寫市場判斷。
- 玩法說明要具體到玩家腦中有畫面：打妖怪、拿隨機強化、蓋房子、種作物、做花束、開電車磨軌。
- 每段盡量避免後台詞：
  - 不寫：`辨識度高`
  - 不寫：`適合放進清單`
  - 不寫：`月推片節奏轉換`
  - 不寫：`玩家對位明確`
  - 改寫成：`如果你喜歡...可以注意`
- 不要過度堆形容詞。像 `舒服`、`療癒`、`荒謬`、`爽` 可以用，但後面要接具體玩法支撐。
- 遊戲名稱不要一直補英文名，除非台灣玩家真的需要靠英文辨識。
- 轉場自然就好：
  - `再來是...`
  - `接下來這款是...`
  - `下一款是...`
  - `最後...`
- 不要把每款都寫成同一個句型，尤其不要每段都開頭 `這款最吸引我的地方`。

### 特別要避開的 AI 感

- `不是...而是...`、`不要...而是...` 不是扣分項，而是阻斷項；正文任何段落出現都要整句改寫，不能只當「不要當主要句型」處理。
- 少用 `最特別的地方`、`最大賣點`、`核心玩法` 這種固定模板；能直接講玩法就直接講。
- 少用「我會推薦給...」這種太工整的結尾；使用者雲端稿更常寫成 `如果你喜歡...這款可以...`。
- 不要把「台灣觀眾會懂」「關注度高」寫進正文，這些只能留在研究判斷裡。

### 2026-06-07 AI 句型漏檢修正

- 不能只在對話裡記規則；月推稿生成後必須跑本地 AI pattern / final draft 檢查。
- `不是...而是...` 類句型會讓稿子變成 AI 在整理對比，正常介紹應直接改成玩家動作或玩法描述。
  - 不寫：`玩家主要做的不是自己下場操作，而是每一輪決定要招募誰。`
  - 改寫：`玩家不需要自己下場操作。每一輪重點是決定要招募誰、隊伍怎麼搭、哪些道具要留下來。`
- 發現抽象評論先行時，不要只刪詞；要回到「玩家看到什麼、玩家做什麼、遊戲接著怎麼變」重新寫。
- 用詞檢查要看中文慣用搭配，不只看意思是否接近；例如不要寫 `材料跑起來`、`基地整理順了`，要改成 `素材穩定累積`、`生產線接起來`、`基地開始自動運作`。

## 這次使用者明確指出的問題

- 不要寫「它會比一般生存建造更有自己的節奏」這種中文不自然的抽象句。
  - 問題：像翻譯腔或企劃評語，不像玫玫會直接念的句子。
  - 改法：改成更具體的玩家感受，例如「玩起來會更舒服」、「探索和整理家園的部分會更明顯」、「會更適合喜歡慢慢蓋家的玩家」。

- 不要寫「前作在台灣其實有一批玩家認識」這種生硬市場判斷。
  - 問題：像研究報告，不像口播。
  - 改法：改成玩家對位，例如「如果你以前玩過前作」、「前作本來就是很適合跟朋友一起刷地城的遊戲」、「喜歡前作那種清版刷寶感的人可以注意」。

- 不要把創作者後台判斷寫進文稿。
  - 禁用例：「那種速度感很適合拿來當月推片的節奏轉換。」
  - 禁用例：「觀眾大概就知道這款為什麼值得放進清單了。」
  - 問題：這是選題或剪輯判斷，不是觀眾會聽的遊戲介紹。
  - 改法：只講遊戲本身的體驗、玩法和適合誰。

## 電車炫客這類作品的正確寫法

- 可講：
  - 「把電車當成滑板，在軌道上磨軌、跳躍、翻轉、接連段。」
  - 「最吸引人的地方，是它把電車做成一種特技動作遊戲。」
  - 「喜歡滑板、跑酷、特技連段，或喜歡創意很強的動作遊戲，可以看一下。」

- 少講或不講：
  - 「這款適合當月推片中段節奏轉換。」
  - 「觀眾會知道為什麼值得放進清單。」
  - 「辨識度高所以值得放進清單。」

## 之後每月推薦文稿的檢查點

- 句子必須是可以直接唸給觀眾聽的口播，不要把研究過程、選題理由、影片編排理由寫進正文。
- 推薦理由要回到玩家體驗：玩起來做什麼、爽點在哪、適合誰。
- 避免抽象萬用詞：「有自己的節奏」、「辨識度高」、「適合放進清單」、「適合月推節奏轉換」。
- 可以保留正面判斷，但要像人講話：具體、直接、貼近玩家。
