# 007 First Light v2 文筆失敗檢驗

日期：2026-05-24

使用者回饋：

- `你上一份文稿還是很爛 我根本不會寫成這樣`
- `重點不是架構 是文字 文筆`

結論：這版不是章節問題，是文字聲音失敗。不能再沿用這版做小修。

## 對照樣本

高可信買前稿：

- `workspace/deliverables/script/2026-04-27-forza-horizon-6-buy-before-user-rewrite.md`
- `workspace/deliverables/script/2026-05-10-forza-horizon-6-buy-before-final-script.md`

這兩份的共同特徵：

- 先從玩家正在吵、正在擔心、正在猶豫的地方開始。
- 事實後面會立刻變成買前判斷。
- 常用「但我要老實講」「這裡要講清楚」「不要聽到...就以為...」這種帶路語氣。
- 抽象結論很少單獨站著，後面一定有具體玩家情境。
- 章節不是每次都用同一種 `適合誰 / 誰先等` 表格收法。

## 失敗 1：起句太像評論稿

v2 問題句：

`《007 First Light》目前看起來，不能直接想成把 47 換成龐德的《刺客任務 4》。比較準確的方向，是有明確劇情推進的第三人稱諜報動作冒險。`

問題：

- 這是評論者在定義類型，不是玫玫在跟觀眾講買前提醒。
- `比較準確的方向`、`第三人稱諜報動作冒險` 都太像媒體稿。
- 觀眾的誤會沒有被說成「買回去會怎樣失望」。

應改方向：

`一看到 IOI 做 007，很多人第一個反應一定是，這不就是《刺客任務》換成龐德嗎？但這個期待要先放低一點，不然買回去很容易覺得跟想像中不一樣。`

## 失敗 2：抽象名詞太多

v2 高風險詞：

- `電影式主線`
- `寬線性潛入段落`
- `平台風險`
- `重玩性`
- `自由度真正看任務段落`
- `完整 007 體驗`

問題：

- 這些詞可以出現在研究卡，但不能直接塞進口播正文當判斷。
- 使用者舊稿會把抽象詞壓到具體畫面後面，例如先講玩家會怎麼開車、怎麼解鎖、怎麼被限制，再講這代表什麼。

改法：

- 不先講 `平台風險`，改講 `PC 玩家這次先不要只看配備，Denuvo 已經寫在 Steam 頁面上，首發效能一定要等實測。`
- 不先講 `電影式主線`，改講 `它會讓你前面偷聽、冒充、用 Q 裝備混進去，後面突然接追車、格鬥或爆炸場面。喜歡這種起伏的人會加分，但想慢慢研究一張關卡的人要先知道節奏不是一直停在潛入。`

## 失敗 3：分流句太工整

v2 常見收法：

`喜歡單人劇情冒險，又想要一點潛行和多路線解法的玩家，可以期待。只想要下一款大型《刺客任務》沙盒的玩家，我會先叫你不要急著衝。`

問題：

- 表面上有分流，但語氣像買前導購表格。
- 每章都這樣收，讀起來會很 AI。
- 使用者可以分流，但通常前面會先把玩家心裡那個誤會講出來，最後自然收成提醒。

改法：

- 先講具體誤會：`如果你買《刺客任務》是為了研究一張地圖、慢慢拆路線、每次都想玩出不同暗殺法，那這款現在不能直接用那個標準期待。`
- 再收判斷：`它比較像把 IOI 擅長的潛入和社交騙術，塞進 007 的電影任務裡。這樣想會比較不容易期待錯。`

## 失敗 4：資訊太乾淨，沒有使用者的判斷摩擦

v2 問題：

- 段落整理得很順，但太像 AI 把研究包梳成十章。
- 缺少使用者舊稿常見的「這件事玩家會分兩派」「這裡不能亂吹」「不要聽到限制就以為整款都鎖死」。
- 沒有把觀眾可能誤解的地方拉出來講。

高可信樣本提醒：

`這個設計一出來，老玩家一定會分兩派。`

`不要聽到手環限制，就以為整款遊戲都被鎖得很死哦。`

`這不是叫你不要買。我的意思是，還沒確認的東西先不要想像成已經做到滿分。`

007 之後要補回這種判斷摩擦，而不是只給平滑說明。

## 失敗 5：語助詞救不了主體語氣

v2 有一些 `哦`、`啦`、`我覺得`，但整體仍不像，原因是主體句型是評論腔。

之後檢查順序要改成：

1. 先檢查起句位置是不是玩家問題。
2. 再檢查事實有沒有變成玩家判斷。
3. 再檢查抽象詞有沒有落地。
4. 再檢查分流句是不是公式化。
5. 最後才補口語詞。

## 下一次 007 重寫硬限制

- 不從 v2 小修，要重開文字。
- 開頭先處理 `這是不是 Hitman 換皮` 的玩家誤會，不先定義類型。
- 每章先寫 300 到 500 字文筆樣張，自檢通過後再擴。
- 交正式稿前，Voice Check 必須列出至少 3 個被打掉的 v2 句子和重寫方向。
