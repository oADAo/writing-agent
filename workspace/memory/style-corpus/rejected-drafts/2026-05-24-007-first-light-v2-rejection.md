# Rejected Draft Note: 007 First Light v2

Date: 2026-05-24
User feedback: `你上一份文稿還是很爛 我根本不會寫成這樣`
Follow-up correction: `重點不是架構 是文字 文筆`

## Verdict

The v2 script must be treated as a failed voice sample, not a draft to continue polishing.

The failure is not only wording. The draft uses the wrong writing posture:

- It speaks like a neutral buyer-advice narrator.
- It explains categories before creating a concrete player situation.
- It overuses conditional audience sorting such as `如果你...`.
- It makes the topic feel like a review framework instead of Maymei guiding viewers through a specific buying anxiety.
- It gives correct facts, but the facts are not transformed enough into Maymei-style player judgment.

## More Precise Failure

The user clarified that the main issue is not structure. The issue is prose.

Even if the chapter order is acceptable, the sentence-level writing does not sound like the user:

- Too many sentences sound like neutral review conclusions.
- Too many abstractions appear as polished labels: `電影式主線`, `寬線性潛入段落`, `平台風險`, `最吸引人的地方`.
- Too many paragraphs use tidy audience segmentation: `如果你是...`, `喜歡...可以期待`, `想要...先等`.
- The prose explains the judgment from outside instead of sounding like the user is thinking through the buying decision while talking.
- The rhythm is too clean and modular; Maymei's accepted buy-before prose has more concrete scenes, community argument, personal emphasis, and "這件事我會這樣看" style movement.

## Do Not Repeat

- Do not start a buy-before script by defining the game category too cleanly.
- Do not make each chapter end with generic audience segmentation.
- Do not use `我會把它看成...` as a substitute for a sharp content angle.
- Do not write from the stance of `this game is X for player type Y`.
- Do not rely on voice scores after the user says the draft does not sound like them.
- Do not treat outline approval as proof that the prose will sound right.
- Do not fix a prose failure by only changing chapter order.

## Preferred Direction From Accepted Buy-Before Sample

The accepted Forza buy-before sample starts from a market argument:

- A visible player doubt or community argument.
- A concrete reason this topic matters now.
- A list of upgrades or facts only after the doubt is clear.
- A personal judgment that names what kind of buyer should care.

The judgment should feel like:

- `這章我會這樣下判斷...`
- `這件事不能直接當亂黑...`
- `還沒確認的東西先不要想像成已經做到滿分...`
- `不要聽到...就以為整款遊戲都...`

Instead of:

- `它適合 X，不適合 Y`
- `可以期待 / 建議觀望`
- `這款最吸引人的地方是...`

## Next Attempt Rule

Before rewriting any 007 script, produce an outline card first and get user approval.

The outline card must include:

- Surface question
- Deeper viewer anxiety
- The first concrete hook
- Chapter-by-chapter Maymei judgment
- What each chapter must not sound like

Do not create another full script until the outline card is approved.
