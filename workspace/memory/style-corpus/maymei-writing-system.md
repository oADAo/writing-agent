# Maymei Chinese Writing System

## Purpose
把既有中文文稿轉成可執行的寫作決策系統，而不是只把樣本當 few-shot。

## Global Enforcement
- 2026-05-13 使用者確認：之後任何文稿都要套用這套全域 Maymei 中文撰寫流程，不只限長影片、買前稿或最終定稿。
- 適用範圍包含長影片腳本、Shorts 文本、標題與封面方向、推薦清單、排行稿、攻略稿、買前必看、文案素材包與正式定稿。
- 即使使用者沒有明講 `跑 maymei finalizer`，只要任務會產出可採用文字，都要自動先做：鎖事實、判片型、抓同片型樣本或歷史錨點、建立寫作卡、AI 文風驗收、機械保底檢查。
- 使用者雲端親改稿與 `User Chinese Override` 仍是最高優先，不能因為流程存在就寫回模型自己的順句方式。

## Runtime Workflow
1. 先從攻略研究包鎖定已確認事實、不可講死、玩家問題。
2. 依片型檢索 3 到 5 篇同類中文樣本。
3. 先寫每章 chapter card：玩家問題、玫玫判斷、可用事實、玩家翻譯、禁用句型。
4. 逐章生成可念稿，不一次爆整篇。
5. AI 人工審稿逐句判斷：資料腔、英文翻譯腔、沒有立場、缺玩家情境都要二修。
6. 最後才跑機械檢查作保底。

## User Chinese Override
- 2026-05-13 使用者明確指定：之後撰寫要使用使用者自己雲端改好的中文撰寫，不要回到模型自己的順句方式。
- 最高優先樣本：`玫玫物語｜六月遊戲推薦｜正式稿｜2026-05-11` 使用者雲端最終稿。
- 使用者改好的版本優先級高於抽象聲線規則、模型舊稿、英文資料順序與一般樣本檢索結果。
- 生成正文前，先把研究事實轉成使用者中文，不要把研究過程、選題理由、素材整理邏輯寫進正文。
- AI 人工審稿時新增一條硬檢查：這段像不像使用者會直接念的中文。只要像研究報告、翻譯稿、企劃說明或商店頁改寫，就要二修。

## User Chinese Style Card
- 先講「這款是什麼」，不要先講市場判斷。
- 句子短一點、直一點、資訊往前推，不要包太多固定模版。
- 每款遊戲通常 2 到 3 段就夠：
  - 第一段：遊戲定位或最醒目的賣點。
  - 第二段：玩法上我們會做什麼，讓觀眾腦中有畫面。
  - 第三段：適合誰、為什麼可以留意，或一句自然的個人期待。
- 可以自然用少量個人感，例如 `這款遊戲我超久以前就放進了願望清單`、`算是我自己蠻期待的一款遊戲`，但不能每款都硬塞。
- 推薦清單轉場用自然中文：`再來是`、`接下來這款是`、`下一款是`、`最後`。不要每款都套同一個開頭公式。
- 收尾句要簡單自然，例如 `可以玩一下哦`、`可以留意一下這款哦`、`很值得注意`、`可以先記起來`。
- 正文只講遊戲本身的體驗、玩法和適合誰。不要講後台選題判斷。
- 避免抽象評語：`辨識度高`、`適合放進清單`、`很有自己的節奏`、`月推片節奏轉換`、`玩家對位明確`。
- 少用企劃詞：`核心循環`、`差異化`、`系統深度`、`玩家對位`。改成白話，例如「白天做什麼、晚上做什麼」、「每次進關會怎麼變」、「如果你喜歡什麼，可以注意」。
- 玩法要具體到有畫面：打妖怪、拿隨機強化、蓋房子、種作物、做花束、開電車磨軌。
- `不是...而是...`、`不要...而是...` 是阻斷句型，不是一般扣分項；正文出現就要退回改寫。
- 生成後必須檢查本地 AI pattern / final draft gate，不能只靠對話中的臨時規則。
- 用詞搭配要符合正常中文與遊戲物件邏輯：素材會累積或被搬運，基地會運作，生產線會串起來或跑起來，不能只因為意思接近就硬湊動詞。
