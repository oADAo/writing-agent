# 玫玫物語文章類型索引

## 用途
這份文件記錄的不是單篇稿怎麼寫，而是 `玫玫物語` 長片文章常見的類型分法。

之後只要匯入新的歷史文章、Google Docs 樣本或最終稿，都先用這份文件判斷它屬於哪一種內容型，再回頭抽風格規則。

## 分類原則
- 一篇文章先給 `1 個主類型`，不要同時塞進兩三個主類型。
- 可以再補 `1 到 3 個副標籤`，例如 `Top 清單 / 新手 / 刷資源 / 機制解說`。
- 分類時先看影片 promise，不只看遊戲名。
- 如果標題和正文 promise 不一致，以正文前言真正承諾觀眾的價值為準。

## 主類型

### 1. 新手開局
- 常見訊號：`新手`、`開局`、`第一天`、`必做`、`10 個技巧`
- 這類文章在解決的問題：
  - 前期最容易踩雷什麼
  - 有哪些東西先知道會少走彎路
  - 第一天或前幾小時最值得先做什麼
- 正文順序常見是：
  - 先講為什麼前期會虧
  - 再講今天最該先做的事
  - 補設定、路線、資源或系統細節
  - 收在「知道這些之後會玩得更順」

### 2. 完整攻略
- 常見訊號：`完全攻略`、`支線任務攻略`、`技巧`、`神殿`、`BOSS`、`機制解說`
- 這類文章在解決的問題：
  - 一個明確關卡、系統、支線、魔物或章節怎麼過
  - 位置、順序、條件、例外是什麼
- 正文順序常見是：
  - 先講這個主題難在哪裡
  - 再拆位置、條件、步驟或機制
  - 補最容易漏掉或做錯的點
  - 收在真正的過關核心

### 3. 資源效率
- 常見訊號：`刷錢`、`刷等`、`掛機`、`效率`
- 這類文章在解決的問題：
  - 哪裡刷最快
  - 前置條件和站位是什麼
  - 要怎麼把每分鐘收益拉高
- 正文順序常見是：
  - 先丟結果與收益
  - 再講地點、條件、屬性或裝備
  - 補優化細節與避免失敗的方法
  - 收在最影響效率的關鍵點

### 4. 配裝流派
- 常見訊號：`流派`、`配點`、`裝備推薦`
- 這類文章在解決的問題：
  - 哪套 build 最好上手
  - 核心技能、裝備、點法為什麼強
  - 適合哪種玩家直接抄
- 正文順序常見是：
  - 先講這套的結果感
  - 再講核心技能或法術
  - 補配點、裝備、操作要領
  - 收在這套最值回票價的理由

### 5. 排行精選
- 常見訊號：`Top 5`、`Top 10`、`排名`、`最難`、`最強`
- 這類文章在解決的問題：
  - 幫觀眾快速看完一串值得比較的對象
  - 順便交代排名標準與每一項的亮點
- 正文順序常見是：
  - 先講這次怎麼排
  - 再逐項講記憶點
  - 補稀有度、取得方式或難點
  - 收在最值得記住的名次或總結

### 6. 推薦清單
- 常見訊號：`推薦`、`必玩`、`大作`、`月份遊戲推薦`、`本月新作推薦`
- 這類文章在解決的問題：
  - 這段時間有哪幾款值得注意
  - 哪些是熱門大作，哪些是私心名單
  - 各自適合哪種玩家
- 正文順序常見是：
  - 先講這次挑選邏輯
  - 再講每款遊戲最有記憶點的賣點
  - 補玩法或氣氛
  - 收一句明確推薦判斷
- 研究規則：
  - 每一款遊戲都要有獨立研究卡，不能只用商店短介或新聞摘要帶過。
  - 每款都要盡量找相關主題 YouTube 影片並抓 `字幕 / 逐字稿`。
  - 官方來源只用來確認基本事實，不可以只靠官方宣傳決定推薦程度。
  - 每款都要補玩家社群來源，包含 Steam 討論、Reddit、巴哈姆特、B 站留言或動態、YouTube 高讚留言、X 或 Threads；實際平台依遊戲生態調整。
  - 如果沒有可用字幕或逐字稿，要標記 `YT 字幕不足`，再用新聞正文、論壇正文、玩家留言或社群討論補強。
  - 推薦理由要回到 `玩法記憶點 / 適合誰 / 為什麼這個月值得提`，不要只寫情緒形容詞。

### 7. 買前必看 / 入坑判斷
- 這是保留類型，專門給未來的 `值不值得買`、`適不適合你`、`該買哪個版本` 這類文章。
- 目前我在本機 `docx` 成稿庫裡，還沒看到足夠強的樣本，所以先保留分類位，不硬把 `新手開局` 混進來。

## 對 agent 的直接用途
- 如果使用者要我「更像玫玫本人」，不能只學句子，還要先判斷是哪一種文章類型。
- 同樣是 `我推薦`，在 `新手開局`、`資源效率`、`推薦清單` 裡的落點不一樣。
- 先判類型，再套 `voice_memory.md` 裡對應的句法和節奏，會比直接拿單篇稿硬模仿穩很多。
