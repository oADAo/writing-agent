# 玫玫物語熱門主題搜尋規則

## 用途

這份文件只負責一件事：替 `玫玫物語` 找出最近值得做的 YouTube 題目。

研究的重點不是整理新聞，也不是只看 YouTube，而是找出：
- 現在做什麼題目最有流量機會
- 哪個角度最適合這個頻道
- 哪些外國已爆的題型還能轉成中文圈可吃的內容

## 先過邊界

正式研究前，先讀：
- `docs/profiles/may-story/channel_scope.md`

如果題材不符合頻道邊界，就不要硬做。

## 必做研究範圍

固定研究三個語圈：
- 中文
- 日文
- 英文

固定研究五種訊號：
- 高流量頻道最近在做什麼
- 單支爆款影片最近在爆什麼
- 這款遊戲最近的官方節點是什麼
- 這款遊戲在高流量社群、論壇、攻略站最近都在討論什麼
- 中文圈目前缺什麼角度

## 搜尋方法硬規則

### 1. 一定要用語言原生關鍵字

同一個主題，三個語圈要分開找，不要只拿同一組英文關鍵字硬套全世界。

常見做法：
- 中文：`遊戲名 + 買前 / 值不值得 / 必做 / 攻略 / 傳說 / 稀有 / 隱藏 / 別做`
- 日文：`遊戲名 + 買い / レビュー / コツ / 後悔 / 伝説 / レア / 面白い`
- 英文：`game + worth it / review / tips / mistakes / legendary / rare / hidden`

### 2. 每次都要查 YouTube 以外的高流量站外來源

熱門主題搜尋每次都要查 `YouTube + 站外來源`，不能只看影片平台。

站外來源不只限於固定名單，而是要依遊戲生態找玩家真的會去看的高流量地方，例如：

- `Reddit`
- `X / Twitter`
- `Threads`
- `巴哈姆特`
- `Steam Community`
- 官方論壇
- Discord 公開討論區
- GameFAQs、wiki、專門攻略站
- 其他和該遊戲高度相關、而且有明顯高流量訊號的站點

原則上至少要查 `2 種站外來源類型`。如果某款遊戲站外生態真的很薄，必須在輸出裡明講資料不足，不可以假裝查過也不可以硬湊結論。

### 3. 先抓主題簇，不先抓單片或單篇爆文

研究時先把影片整理成幾個 `主題簇`，例如：
- 買前評價
- 新手必做 / 錯誤
- 傳說 / 稀有 / 隱藏解鎖
- 好笑事故 / 反差時刻

站外來源也一樣，要先整理成幾個 `討論簇 / 問題簇 / 攻略需求簇`，再判斷哪個題型在反覆出現。

先判斷哪個主題簇在反覆出現，再看每個主題簇裡哪幾支影片、哪幾篇高流量討論最強。

### 4. 有效熱度判定

只有符合下列其中一條，才算真正熱門題型：
- 同主題在至少 `2 個語圈` 都反覆出現
- 同語圈有 `2 位以上` 高流量創作者都在做，而且觀看量明顯不低

如果要把題目升格成主結論，還要再通過下列其中一條跨來源驗證：
- `YouTube` 在至少 `2 個語圈` 都反覆出現，而且站外至少 `1 個` 高訊號來源也在討論同一題型
- 站外至少 `2 個` 高訊號來源都反覆出現同一題型，而且 `YouTube` 至少有 `1 個` 強主題簇能對上

高訊號來源不是只看有沒有提到，而是要看它在該平台是否真的有明顯互動或熱門表現，例如：
- 高按讚 / 高愛心
- 高留言 / 高回覆串
- 高轉發 / 高分享
- 高收藏 / 高書籤
- 站內熱門 / 置頂 / 長串討論
- 被其他玩家或其他平台反覆引用的攻略頁

如果只是：
- 一支低觀看 Shorts
- 一支 VTuber 臨場反應
- 一支 bug / 梗圖 / 事故片
- 一篇單次偶發爆文

但其他語圈和其他創作者都沒有跟進，就先當 `特例`，不要硬推成主要題目。

### 5. 查詢證據硬規則

每次研究都要留下可驗證的查詢證據，至少要交代：

- 查了哪些平台或站點
- 每個平台用了哪些語圈原生關鍵字
- 找到哪些高流量貼文、討論串、攻略頁或站內熱門頁
- 哪些來源被納入主結論，哪些只是補充案例
- 哪些地方資料不足，還不能下重結論

如果輸出裡沒有查詢證據，就不算完成這一步。

### 6. 如果研究目標是 Shorts，先看高流量 Shorts

找 Shorts 題型時，優先直接找 `真正有流量的 Shorts`，不要先拿長影片主題硬套。

但同時要注意：
- 不要把 `秒數短` 當成 `Shorts` 證據
- 不要把普通短片、切片或低觀看 Shorts 當成主市場訊號

如果某款遊戲的 Shorts 生態真的很薄，或高流量 Shorts 樣本太少：
- 才回到三語圈整體內容市場補驗證
- 補驗證完，再把主題壓成 Shorts punch

## 主題研究流程

### 1. 先確定研究對象

- 遊戲名稱
- 平台
- 發售時間或近期更新節點
- 目前處於哪個流量階段：
  - 發售前
  - 發售初期
  - 熱度延續期
  - 更新回溫期

### 2. 掃跨語圈熱門內容

至少觀察：
- 中文圈近期高流量影片
- 日文圈近期高流量影片
- 英文圈近期高流量影片

每個語圈要看的不是只有題目，而是：
- 標題切角
- 封面資訊
- 影片主題類型
- 觀眾會點的理由
- 這支片屬於哪一個 `主題簇`

### 3. 掃高流量社群與站外攻略討論

至少觀察：
- 哪些高流量貼文、討論串或攻略頁反覆在講同一件事
- 哪些問題明顯是玩家最近最常問、最常卡、最常搜尋的
- 哪些站外高流量討論和 YouTube 題型能互相對上
- 哪些只是單篇爆文、單次梗圖或一次性事故，不應升格成主題

### 4. 判斷哪些題型能翻成中文圈可吃角度

研究時要特別找這幾種：
- 外國已經明顯爆，但中文圈還沒有大量跟進
- 中文圈有人做，但切角很弱
- 很有搜尋需求，而且能做成長尾內容
- 符合 `玫玫物語` 熟悉題型的高實用內容

### 5. 回到頻道適配度

最後不是選「全網最紅」，而是選：
- 這個頻道做起來最像
- 最容易寫出好腳本
- 最容易做出高點擊標題與封面

## 題目評分標準

每個候選題目都要從下面幾個角度判斷：

- `頻道適配度`
  - 像不像 `玫玫物語` 本來就會做的內容
- `熱度訊號`
  - 最近是否真的有明顯熱度或需求
- `中文圈可轉譯性`
  - 外國爆款角度翻成中文後會不會自然
- `實用收益`
  - 題目是否能明確講出觀眾得到什麼
- `長尾價值`
  - 這支影片是不是只吃一波，還是有機會持續搜尋

## 優先來源

### 第一層：官方與平台資訊
- 官方網站
- 官方公告
- 商店頁面
- 發售與更新資訊

### 第二層：高流量競品內容
- 中文高流量頻道
- 日文高流量頻道
- 英文高流量頻道

### 第三層：高流量社群、論壇與攻略站
- 社群平台上的高互動貼文與討論串
- 論壇上的長串熱門討論
- 攻略站、wiki、FAQ 或常被引用的教學頁
- 該遊戲社群真正會去查攻略與交流的站外來源

### 第四層：搜尋需求訊號
- 搜尋結果裡反覆出現的問題
- 買前、開局、踩雷、效率、隱藏要素等高需求問題

## 研究時一定要回答的問題

- 這個題目現在做，流量理由是什麼
- 如果要模仿外國爆款，它的核心角度到底是什麼
- 這個角度是 `跨語圈重複出現`，還是只是 `單支特例`
- 除了 YouTube，哪些高流量社群、論壇或攻略站也在反覆討論這個題型
- 我實際查了哪些地方、用了哪些關鍵字、看到哪些高訊號證據
- 中文圈目前缺的是哪一個切法
- 這個題目最後能不能自然延伸成：
  - 高點擊標題
  - 封面文案
  - 長影片腳本

## 禁止行為

- 不要只列出遊戲功能點，沒說流量理由。
- 不要只看中文圈就下結論。
- 不要只抓到一支片就說這是熱門主題。
- 不要只抓到一篇高互動貼文就說這是熱門主題。
- 不要把低觀看 Shorts 或小圈子切片直接當成全市場訊號。
- 不要說有查社群、論壇或攻略站，卻不附查詢證據。
- 不要用錯語言關鍵字後，再拿錯誤結果硬下判斷。
- 不要把很多只是改寫字眼的題目當成不同選項。
- 不要明明資料不足，還硬推一個題目。
- 不要把不符合頻道邊界的熱門題目塞進建議。

## 一句話版交接

熱門主題搜尋不是找「現在最紅的全部東西」，而是先用三語圈原生關鍵字找出反覆出現的 `主題簇`，再把 `YouTube + 高流量社群 / 論壇 / 攻略站` 的訊號交叉比對，排除單支特例片與單篇爆文，最後找出 `玫玫物語` 現在最值得做、又最有機會在中文圈吃流量的題目。
