# Topic Research External Signal Design

**Date:** 2026-03-19

## Goal

把 `玫玫物語` 的 `熱門主題搜尋` 從「以 YouTube 為主、社群只當補充」改成「每次都必查 YouTube 以外的高流量社群 / 論壇 / 攻略站，並留下查詢證據」的正式流程。

## Problem

目前文件有提到 `社群與搜尋訊號`，但規則太泛：

- 沒有寫清楚每次研究是否必查
- 沒有要求依遊戲生態動態選平台
- 沒有定義什麼叫高流量、高訊號來源
- 沒有要求輸出查詢證據
- 沒有要求把 YouTube 與站外訊號交叉驗證後再下主結論

這會讓執行結果很不穩定，容易出現：

- 只看 YouTube 就下結論
- 只拿單篇社群爆文硬推主題
- 查過哪些地方無法被驗證
- 結論缺少跨來源佐證

## Goals

- 每次 `熱門主題搜尋` 都必查 YouTube 以外的高流量訊號來源
- 來源不寫死成少數平台，而是依遊戲實際生態動態選擇
- 要求研究者留下可驗證的查詢證據
- 主結論必須建立在 YouTube 與站外來源的交叉驗證上
- 輸出格式要明確看得出查了什麼、為何納入、哪些只是補充

## Non-Goals

- 不建立固定平台白名單
- 不為每個平台定義單一數字門檻
- 不自動化資料抓取或 API 整合
- 不修改標題、腳本、Shorts 的工作模式

## Recommended Approach

採用「結構化但保留彈性」的更新方式：

1. 在 `AGENTS.md` 補上共通規則與模式 1 規則
   - 熱門主題搜尋除了 YouTube，也必查高流量社群 / 論壇 / 攻略站
   - 查詢平台要依遊戲生態選，不限於 `Reddit / X / Threads / 巴哈`
   - 研究結果必須附查詢證據

2. 在 `topic_research_rules.md` 補完整執行規格
   - 固定訊號層級
   - 高訊號來源判準
   - 查詢證據紀錄要求
   - YouTube 與站外來源的交叉驗證規則
   - 資料不足時的處理方式

3. 在 `prompts/topic-research.md` 同步成可直接執行的 prompt
   - 明講每次必查
   - 明講要列出查詢平台、關鍵字、代表高流量內容
   - 明講主題判定必須跨來源驗證

## Source Selection Rules

站外來源不只限於固定社群平台，而是優先找：

- 該遊戲玩家真的會討論攻略、買前、踩雷、效率玩法的地方
- 該遊戲有高互動、高回覆、高收藏、高轉發或站內熱門表現的地方
- 常被搜尋結果、玩家回文或其他平台反覆引用的地方

可用來源示例：

- `Reddit`
- `X / Twitter`
- `Threads`
- `巴哈姆特`
- `Steam Community`
- 遊戲官方論壇
- Discord 公開討論區
- GameFAQs / wiki / 專門攻略站
- 其他和該遊戲高度相關、且有明顯高流量訊號的站點

## Evidence Requirements

每次研究都要留下至少這些證據：

- 查了哪些平台 / 站點
- 每個平台用了哪些語圈原生關鍵字
- 找到哪些高訊號貼文、討論串、攻略頁或熱門頁
- 哪些來源被納入主結論，哪些只當補充案例
- 哪些點仍缺證據，必須標成 `Unknown`

## Cross-Source Validation Rule

主結論不能只靠單一來源成立。

推薦題目至少要滿足下列其中一條：

- `YouTube` 在至少 2 個語圈都反覆出現，且站外至少 1 個高訊號來源也在討論同一題型
- 站外至少 2 個高訊號來源都反覆出現同一題型，且 YouTube 至少有 1 個強主題簇能對上

如果只有單篇爆文或單支影片，最多只能當補充案例，不可直接升格成 `Top 1 Recommendation`。

## Output Changes

`Topic Brief` 需要新增可驗證欄位：

- `## Query Log`
- `## Community / Forum Signals`
- `## Cross-Source Validation`

每個候選題目也要說清楚：

- `YouTube 證據`
- `站外證據`
- `為什麼有流量`
- `為什麼適合這個頻道`

## Acceptance Criteria

- `AGENTS.md`、`topic_research_rules.md`、`prompts/topic-research.md` 的規則一致
- 三份文件都明確要求每次必查站外高流量來源
- 三份文件都明確要求輸出查詢證據
- 三份文件都明確要求 YouTube 與站外來源交叉驗證
- 文件保留平台彈性，不把來源限制在使用者列的範例平台
