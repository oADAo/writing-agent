# Monthly Game Recommendations Workflow

這份文件是 `月份遊戲推薦 / 六月遊戲推薦 / 本月新作推薦 / 值得期待的新遊戲` 的專用流程。

固定分三段：

1. `研究`
2. `撰寫`
3. `提交`

不要跳步。除非使用者明確說「直接寫完整稿 / 最終定稿 / 可直接朗讀版」，否則先交研究素材包。

## 固定雲端位置

- Google Drive 資料夾：`https://drive.google.com/drive/u/0/folders/1yFgnPkH9SLdhSk3fGiEMxySkpAwXEGDy`
- 文案：使用資料夾中的月份推薦 Google Docs 文案檔。
- 表格：使用同資料夾中的 `Longform Episode Workbook Template`。
- 工作表：`遊戲清單`

`遊戲清單` 欄位固定：

| 欄位 | 名稱 |
| --- | --- |
| A | 遊戲名稱 |
| B | 遊戲宣傳片 |
| C | 遊戲實機影片 |
| D | 遊戲音樂 |
| E | 發售日期 |
| F | 平台 |

## 第一段：研究

目的：先確定哪些遊戲值得進名單，以及每款能不能講得出真實賣點。

必做：

- 建立候選遊戲池。
- 確認平台、發售日、是否符合 `PC / Switch / PS5` 與買斷制邊界。
- 每款都是獨立研究單位，不用一句商店介紹帶過。
- 官方來源只負責確認發售日、平台、版本、玩法資訊與已公開內容。
- 每款都要補非官方或半非官方訊號，例如：
  - YouTube `gameplay preview / demo impression / before you buy / review / hands-on / trailer breakdown`
  - YouTube 月份推薦 / 新作推薦：`July games / upcoming games / new games / best games this month / PC Switch PS5 games July`
  - 中文 `試玩心得 / 值不值得期待`
  - 巴哈姆特、Steam 討論、Reddit、B站留言或動態、YouTube 高讚留言、X 或 Threads
- 每款都要盡量找至少 `2 支` 非官方創作者影片；其中至少 `1 支` 優先找「7 月遊戲推薦 / upcoming games / new games」類影片，確認其他人是否把這款放進同月推薦，以及他們實際怎麼介紹。
- 高訊號影片必須依全專案 `source-capture` 規則抓字幕、逐字稿、本地轉錄或可保存留言正文；抓不到文字證據時，不得把該影片當作主證據，必須改找其他可保存文字來源或標成資料不足。
- 網站、論壇、新聞、商店頁要讀正文並保存原文 / 正文擷取結果，不只看搜尋摘要。

研究卡固定包含：

- 已確認事實
- 核心玩法
- 最大賣點
- 社群期待與疑慮
- 其他創作者介紹時實際講了什麼
- 適合誰
- 風險或未定點
- 可直接改寫口播句

研究階段交付：`月份推薦研究素材包`。

## 第二段：撰寫

目的：把使用者確認的名單，收斂成玫玫可以直接念的正式稿。

必做：

- 使用者已確認的名單、刪改與順序是鎖定條件。
- 遊戲順序固定依發售日期由早到晚。
- 套用 `maymei-script-finalizer+ 中文撰寫系統`。
- 先學使用者雲端親改稿的句長、轉場、收尾與判斷語氣。
- 遊戲名稱盡量用台灣玩家看得懂的中文譯名，必要時第一次出現補英文名。

文案格式固定：

```md
影片標題

大家好~這裡是玫玫物語。

## 遊戲名稱

正文...

## 下一款遊戲

正文...

自然訂閱、留言與收尾句。
```

正文要講：

- 這款是什麼
- 玩家實際會做什麼
- 最大玩法記憶點
- 適合誰
- 哪些地方要保守講，不要講死

正文不要寫：

- 研究過程
- 選題理由
- 表格填寫邏輯
- 後台判斷
- 查證報告腔

## 第三段：提交

目的：把正式稿與影片素材表放進固定雲端位置，並確認真的寫入。

預設提交兩份：

1. Google Docs 文案檔：寫入可直接朗讀的完整稿。
2. Google Sheets `遊戲清單`：更新宣傳片、實機、音樂、發售日期與平台。

除非使用者明確說只改其中一份，否則每月遊戲正式交付預設包含 `文案 + 宣傳片 / 實機 / 音樂表`。

寫入後必須重新讀取：

- Google Docs 目標內容
- Google Sheets `遊戲清單` 目標範圍

確認成功後再回報。

## 表格搜尋規則

### 遊戲宣傳片

搜尋順序：

1. `<遊戲英文名> official trailer`
2. `<遊戲英文名> announcement trailer`
3. `<遊戲英文名> release date trailer`
4. `<遊戲英文名> launch trailer`

優先官方頻道或可信遊戲頻道。

### 遊戲實機影片

每款都要補，不能只放 trailer。

搜尋順序：

1. `<遊戲英文名> gameplay`
2. `<遊戲英文名> demo gameplay`
3. `<遊戲英文名> hands-on`
4. `<遊戲英文名> no commentary`
5. `<遊戲英文名> preview gameplay`

必須是真正看得到實機遊玩的影片，不要用純 CG、純剪輯宣傳片或只有口播新聞的影片代替。

### 遊戲音樂

音樂欄要保守，能空白就不要硬填。

搜尋順序：

1. `<遊戲英文名> soundtrack instrumental`
2. `<遊戲英文名> ost no vocals`
3. `<遊戲英文名> bgm instrumental`
4. `<遊戲英文名> main theme`
5. `<遊戲英文名> menu theme`

優先官方、發行商、開發商、可信音樂廠牌，或明確標示為該遊戲 OST / BGM 的來源。

如果本作音樂找不到，可以依序改找：

1. 系列前作
2. 同開發商或發行商舊作
3. 同類型遊戲純音樂

使用替代音樂時，工作記錄要標明：

- `本作音樂`
- `系列前作`
- `同公司舊作`
- `同類型替代`

不要把替代曲誤寫成該款遊戲官方音樂。

寫入表格前必須實際播放聽過一次，確認：

- 沒有明顯人聲
- 沒有歌詞
- 沒有旁白
- 沒有尖銳音效
- 沒有過吵鼓點
- 適合當背景段落

避開：

- `lyrics`
- `vocal`
- `cover`
- `remix`
- `music video`
- 一般情境音樂
- 粉絲二創
- 不明來源

如果無法實際聽過，或聽完不能確認是乾淨純音樂，直接留空。

## 回報格式

完成後簡短回報：

- 文案是否已更新
- 表格是否已更新
- 是否已回讀確認
- 哪些音樂是本作，哪些是替代來源
- Google Docs / Sheets 連結
