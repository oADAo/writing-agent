# Source Capture Research Rules

這份是全專案研究硬規則，不限於每月遊戲推薦。只要任務需要查資料、判斷熱度、整理玩法、寫研究包、準備標題封面、寫長片 / Shorts 文案前置資料、做業配或任何內容研究，都要套用。

## 核心原則

- 研究結論只能建立在已讀到、且能保存或回查的文字證據上。
- 搜尋結果、新聞摘要、影片標題、縮圖字、商店短介、AI 摘要、Google snippet、社群貼文列表，只能當線索，不能當主結論證據。
- 主結論來源必須至少符合其中一種：
  - `opencli web read` 或等價工具讀到的網頁正文。
  - YouTube / bilibili / TikTok / Reels 的字幕、逐字稿、本地音訊轉錄。
  - 論壇、留言、社群討論的可保存正文或截圖 OCR。
  - 官方頁、新聞頁、攻略頁、商店頁的正文擷取結果。
  - 使用者提供的截圖、文件、附件，搭配 OCR 或人工摘錄。
- 沒有抓到正文、字幕、逐字稿、留言文字、OCR 或本地轉錄，就不得把該來源當成深度研究證據。

## 社群 / 創作者影片硬規則

官方影片只能確認官方想講什麼，不能代表玩家或市場怎麼理解。研究時必須另外抓 `非官方創作者 / 社群影片`，用來回答：其他人介紹這款遊戲時，到底講了什麼、怎麼包賣點、提醒哪些疑慮。

適用來源包含：

- YouTube 月份遊戲推薦：`July games`, `upcoming games`, `new games`, `best games this month`, `PC/Switch/PS5 games July`。
- 遊戲個別介紹：`gameplay preview`, `demo impression`, `hands-on`, `before you buy`, `trailer breakdown`, `things to know`。
- 中文 / 日文 / 英文創作者介紹影片；有 bilibili 生態時也要找 B 站影片或留言。
- 小型或中型創作者優先於官方、發行商與大型媒體通稿；官方片只能當事實校正。

最低要求：

- 研究清單或月份推薦時，每款遊戲都要盡量找至少 `2 支` 非官方創作者影片；其中至少 `1 支` 優先找「月份推薦 / upcoming games」類影片，看它是否被其他人放進同月推薦。
- 單款買前必看 / 攻略 / 深度研究時，至少找 `3 支` 非官方創作者影片或同等玩家實測來源。
- 每支納入結論的影片必須抓到字幕、逐字稿、本地音訊轉錄、影片說明、置頂留言、高讚留言，或其他可保存文字證據。
- 研究包必須整理「創作者實際說了什麼」，不可只列影片標題、觀看數、發布時間。

抓不到時：

- 如果找得到影片但抓不到字幕 / 逐字稿 / 留言文字，該影片只能列為候選線索，不得支撐結論。
- 必須改找同主題其他創作者影片、其他語言版本、B 站 / YouTube 留言、或能保存文字的 preview / hands-on 文章。
- 如果某款遊戲真的找不到足夠社群影片文字證據，研究包要標 `社群影片文字證據不足`，不能寫成「社群都在期待」或「大家都推薦」。

## 必做保存

每次研究 run folder 至少要有：

- `query-log-reviewed.md`：平台、語言、關鍵字、指令、命中、是否納入。
- `sources.md`：每個來源支撐哪個結論，並標明來源層級。
- `creator-video-evidence.md` 或同等章節：保存非官方創作者 / 社群影片的字幕、逐字稿、轉錄、留言文字與重點摘要。
- `source-originals/` 或同等資料夾：保存讀到的原文 / 正文擷取 / 字幕 / 逐字稿 / OCR / 留言文字。
- `tool-readiness.md` 或等價工具紀錄。
- 研究成功後的 zip package 與 `PACKAGE-MANIFEST.md`，說明哪些原文已保存、哪些抓不到、哪些來源只作線索。

## 抓不到時的處理

如果某個來源抓不到正文或字幕，不能直接用它寫結論。先依序嘗試：

1. 換 `opencli web read --url <url>`，必要時加 `--wait`、`--wait-until networkidle`、`--diagnose`。
2. 對影片先抓官方字幕，再抓自動字幕，再抓其他語言字幕。
3. 字幕抓不到時，用可行工具抓音訊做本地轉錄。
4. 音訊也不可得時，改抓影片說明、置頂留言、高讚留言、創作者貼文或同主題其他影片的逐字稿。
5. 論壇或社群頁讀不到時，換手機版、列印版、快取、同串其他頁、站內搜尋結果後再讀正文；巴哈、Reddit、Steam Community、官方論壇都照這個原則。
6. 官方頁讀不到時，改找同發行商新聞稿、平台商店頁、官方 YouTube 說明欄、官方 X / news post 的可保存文字。

仍然抓不到時：

- 在研究包寫 `未能擷取原文 / 字幕 / 正文`。
- 降低或移除該來源權重。
- 改找其他可保存文字證據。
- 如果沒有替代證據，該點只能列為 `資料不足 / 待查證`，不能寫成結論。

## 研究包寫法

每個被納入結論的來源都要能回答：

- 我實際讀到了哪一段正文、字幕、留言或攻略內容？
- 這段內容支撐哪個事實、玩家疑慮、玩法判斷或推薦理由？
- 這是官方確認、玩家回報、創作者實測、媒體整理，還是研究推論？
- 原文或擷取檔存在哪裡？

正式研究包不能只列網址。`Sources` 或 `來源重點對照表` 必須寫出每個來源實際支撐的結論。

## 禁止事項

- 禁止用搜尋結果摘要補出玩法細節。
- 禁止只看影片標題、縮圖或觀看數就寫「玩家都在討論」。
- 禁止把抓不到字幕的高流量影片當成主證據。
- 禁止把單篇新聞或官方商店短介當成玩家需求證據。
- 禁止為了讓研究包看起來完整，把資料不足的點寫成肯定句。
- 禁止研究成功後只交 markdown，不打包原文與證據。
