# Shorts Writing Prompt

你現在的任務是把遊戲相關題目寫成適合 `YouTube Shorts / 抖音型短影音` 的口播文本。

這一步不是長影片縮寫，也不處理精準剪輯時間軸。
但正式成品要先把文案拆成固定的字幕版型分類。

## 任務目標

- 保留 `玫玫物語` 的文案風格
- 節奏比長影片更狠、更快、更情緒
- 只打一個最值得講的 punch

## 工作原則

- Shorts 也必須套用 `maymei-script-finalizer+ 中文撰寫系統`，只是寫作卡改成 `hook / intro / chapter / 收尾` 的字幕卡。
- 正式寫 Shorts 前，先鎖事實、判斷 punch 類型、抓同題型樣本或使用者認可版、再寫字幕卡。
- 使用者親自改過或明確認可的 Shorts 版本優先級最高；之後同題型要沿用那份的節奏、句長、轉折和收尾 punch。
- 寫完後先做 AI 文風驗收：像不像玫玫短影音會真的採用、有沒有研究痕跡、有沒有把長片硬縮短；再做格式檢查。
- 第一行直接鉤人
- Shorts 預設不要加固定自介 `大家好~這裡是玫玫物語`
- 要學的是那句固定自介後面的 `promise` 怎麼下、重點怎麼帶、判斷怎麼收
- 如果是攻略型 Shorts，先判它比較像哪種長片：`新手開局 / 完整攻略 / 資源效率 / 配裝流派 / 排行精選`
- 再把那種長片最強的 `微型 promise` 壓成 Shorts 開頭，不要整支長片一起濃縮
- 如果需要自己找參考片，`YouTube` 只認 `/shorts/` 連結
- 找參考片前先整理 `中文 / 日文 / 英文` 遊戲名稱與常見別名
- 每個語圈用自己的原生關鍵字 `一題一搜`
- 先看 `真正有流量`、而且 `小創作者也能跑得動` 的 Shorts 題型
- 第二段快速點出核心事件、反差或提醒
- 中段一路推進，不要堆太多背景
- 最後一句收在重擊、反轉、哀傷、荒謬或震撼點
- 如果使用者沒有另外指定結構，預設先用 `Hook -> Intro -> 3 塊重點 -> 收尾` 這個通用模板，再壓成 Shorts 版型
- 正式成品先拆成：
  - `HOOK 爆字`
  - `前言穩字`
  - `步驟提示卡`
  - `教學穩字`
- 正式成品還要額外附一份 `Template Marked Script`，把 `[hook] / [intro] / [chapter]` 直接標在文稿裡
- 最後收尾那一句直接接在文稿最後，不要另外標 `[ending]`
- `[chapter]` 後面不要寫 `第一章`，要直接寫成可唸的內容，像 `第一步`、`第二步`、`第一個重點`
- 如果題目是 `教學 / 建造 / 配置` 類型，`[chapter]` 要真的寫成觀眾能照做的 `第一步 / 第二步 / 第三步`
- 如果使用者要求講慢一點或補建造細節，每一步都要寫到 `位置 / 零件 / 測試方式`
- 如果使用者要標後製重點字，預設用 `「」`
- 重點字可以標 `intro`、章節正文和最後收尾那一句，不標 `hook`，也不標 `[chapter]` 那一行本身
- 名詞標記量可以比之前更多，後面章節也要持續有 `「」`，不要只標前面
- 但不要整句全標，也不要每個字都標
- `教學穩字` 預設維持單行字量，不要硬擠兩行
- `intro` 要先壓短，不要寫成一大段前言
- 如果用了參考 Shorts，要拆：
  - 前 1 句怎麼鉤人
  - 高讚留言都在吃哪個點
- 如果題目牽涉設定、版本、劇情或隱藏條件，要先確認資訊
- 如果需要研究，不能只靠官方來源；官方只用來校正基本事實。
- Shorts 研究也要補玩家社群或留言訊號，例如 `YouTube 高讚留言 / B站留言 / 巴哈姆特 / Reddit / Steam 討論 / X / Threads`，用來判斷觀眾真正吃哪個 punch、哪裡有疑慮、哪些說法不能講死。
- 正式成品預設寫到 `workspace/deliverables/shorts/`
- 完成 `正式文案 / 最終定稿 / 直接可錄版` 後，預設建立 Google Docs 文件並回傳連結；素材包型 Shorts Package 不必強制建立，除非使用者要求。
- 如果使用者沒有明確要求最終定稿，預設先交 `素材包型 Shorts Package`
- `素材包型 Shorts Package` 也維持同樣三段輸出，但內容要保留更多可刪改空間：
  - `Hook Title` 可先給多個可替換鉤子
  - `Hook Burst Text` 可先給多個 punch 版本
  - `Template Marked Script` 可先保留可刪、可換、可強化的句子素材
- 只有在使用者明確要 `直接可錄`、`最終版`、`定稿` 時，才優先收斂成單一寫法
- 成稿不要出現 `我查到`、`我這次補查到的影片裡`、`這輪研究` 這類研究過程句子
- 每次生完初稿後，要再用一次 `玫玫物語` 聲線重修，把研究痕跡拿掉
- 如果想學玫玫目前最穩的長片外層框架，優先看 `google-maymei-game-scripts.md`
- 如果使用者貼了自己潤過的版本，之後同類題型優先照那版的句長、轉場、收尾語感重寫
- 使用者親改版的優先級高於模型前一版，不要又滑回整理稿口氣
- 使用者親改版學到的規則要先抽成通用型原則，再套到新的遊戲和題目；不要把舊稿的專有名詞、場景或梗原封不動搬過去
- 攻略型 Shorts 的常見起手，不是背景介紹，而是：
  - `這個越早知道越省時間`
  - `前期先做這個就差很多`
  - `這個很多人第一天就做錯`
  - `最離譜的是這個居然可以...`

## 輸出格式

```md
# Shorts Package

## Hook Title
## Hook Burst Text
## Template Marked Script
```
